Shared sizing
Same row math, flexible width
Preferred width chooses the column count; rendered width stretches to fill the viewport.
Windowed loading
Prefetch stays close to view
The grid can request nearby ranges before the user hits empty space.
Header
Extra content shares the scroll flow
The optional header is part of the same scrolled surface as the virtualized cards.
Crawlable SSR
Real pagination for crawlers
offset and limit describe a repeatable HTML window while the client still upgrades to free scrolling.
DataGrid DSL
The control owns virtualization; the renderer only describes one card.
scala
dataGrid[Post] { items = posts itemWidthPx = 320 itemHeightPx = 180 gapPx = 16 crawlable = true cellRenderer = { (post: Post | Null, index: Int) => div { classes = Seq("post-card") text = if (post == null) s"Loading $index" else post.title } } header { div { text = "Scrolling grid header" } } }
Remote loader
The loader only needs to answer index/limit requests with totalCount and nextQuery.
scala
ListProperty.remote[Post, PageQuery]( loader = ListProperty.RemoteLoader { query => fetchPosts(query.index, query.limit) }, initialQuery = PageQuery(index = 0, limit = 24), rangeQueryUpdater = Some((query, index, limit) => query.copy(index = index, limit = limit) ) )
Crawlable window
The server renders a stable slice, while the browser restores scroll position from the same query.
text
/data-grid?offset=0&limit=24 renders items 0 to 23 /data-grid?offset=24&limit=24 renders items 24 to 47 Browser: scrollTop starts at the row that contains offset SSR: More link points to offset + limit
