Lexical as an SSR-safe client island.
Lexical island
The editor renders content immediately and only becomes Lexical in the browser.
JFX2 treats the editor as a ClientSideComponent: SSR gets a stable fallback with the same shell layout, hydration replaces the surface with Lexical, and external property changes flow back into the editor.

The writing surface activates on the client
Changes are also auto-saved in the browser, so you can restore the last editor state after a reload.
SSR fallback contract
What is rendered on the server must structurally match the first client render.
Shell stays stable
Fallback and client version use the same jfx-editor shell with toolbar area and surface frame.
Toolbar does not flicker
The editor shell stays stable, so hydration can activate Lexical without layout surprises.
Text stays visible
Plain text or Lexical JSON is extracted into preview text during SSR and then adopted by Lexical after hydration.
External values
valueProperty updates are synchronized back into Lexical after mount via parseEditorState.
